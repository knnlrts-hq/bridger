Bridger XG5 Integration POC

End-to-end demo of LexisNexis Bridger Insight XG5 sanctions screening integrated with a corporate payment factory processing pain.001.001.03 files.

All Bridger API endpoints and webhooks are mocked in-browser. No backend required.

Demo Tools

1

pain.001 Parser & Viewer

Upload or paste ISO 20022 pain.001.001.03 XML. Parse and visualize payment instructions with all screenable parties extracted and highlighted.

ISO 20022 XML Parsing Party Extraction
2

Bridger API Explorer

Interactively construct and fire requests against the mock Bridger XG5 REST API. Explore Token, Search, Results, and ListMaintenance interfaces.

REST API Token/JWT Lists/Search Results
3

Screening Flow Simulator

The main event. Load a pain.001 file, watch the full pipeline: parse, authenticate, screen, apply thresholds, assign Trax statuses. Animated end-to-end.

E2E Pipeline Thresholds Trax Statuses
4

Webhook Simulator

Two-panel view: sign webhooks on the Bridger side, verify on the Bank Connector side. Step-by-step HMAC-SHA256 computation. Tamper mode to prove integrity.

HMAC-SHA256 AlertStateClosed AlertDecisionApplied
5

Compliance Review Dashboard

Compliance officer workflow. Review SUSPECT alerts, compare payment parties against watchlist matches, make accept/reject/escalate decisions. Full audit trail.

Alert Review Decisions Audit Trail Webhooks
